## 2. Deskripsi Aplikasi / Project

### Masalah yang Diselesaikan

Banyak reseller produk digital (akun streaming, voucher game, top-up, dsb.) kesulitan membangun toko online sendiri karena:
- Tidak punya kemampuan coding
- Mahal jika harus bayar developer
- Platform marketplace umum memotong komisi besar dan tidak bisa dikustomisasi
- Tidak ada sistem manajemen stok otomatis untuk produk digital

### Solusi: ArufKuy Store

ArufKuy Store adalah **platform e-commerce statis khusus produk digital** yang sepenuhnya dibangun dengan AI. Toko ini menyelesaikan masalah di atas dengan cara:

- **Katalog produk** yang dinamis dari Firebase Firestore dengan live search dan filter kategori
- **Sistem stok otomatis** — stok terpotong sendiri setiap kali transaksi sukses, tanpa intervensi manual
- **Integrasi pembayaran Mayar** — mendukung e-wallet, transfer bank, dan metode pembayaran populer Indonesia
- **Preview Invoice** — halaman konfirmasi pesanan (`detail-invoice.html`) sebelum pembayaran, menampilkan ringkasan lengkap (produk, data pembeli, subtotal, fee, diskon)
- **Detail Order** — halaman tracking pesanan (`detail-order.html`) dengan sistem akses Master Key + Time Window 6 jam
- **Admin Panel lengkap** — CRUD produk, kelola stok, monitoring pesanan, semua tanpa coding
- **Sistem Biaya Layanan (Service Fee)** — kustomisasi fee per produk/varian (mendukung desimal) yang dihitung aman dari server
- **Serverless backend** via Cloudflare Worker — aman, cepat, gratis
- **Konfigurasi terpusat** via `config.js` — satu file untuk semua konfigurasi Firebase dan Worker URL
- **Modul JavaScript terpisah** — `utils.js`, `frontend-features.js`, `admin-features.js`, `theme-engine.js` untuk maintainability
- **Fitur kustomisasi tema** — seasonal themes (Lebaran, Natal, dll) dengan efek partikel
- **Sistem kupon diskon** dan popup pengumuman
- **Auto-cancel pesanan** yang sudah terlalu lama pending
- **Halaman pendukung profesional** — About Us, Contact, Help Center, Order Guide, Feedback, Privacy Policy, Terms of Service, Disclaimer
- **UI/UX profesional** — ikon SVG konsisten, desain dark mode premium, animasi halus, responsive design

Dengan ArufKuy Store, siapa pun bisa punya toko produk digital profesional hanya dengan deploy ke Cloudflare Pages + Firebase, tanpa bayar server.

## 4. Ceritakan Proses "Vibecoding" Kamu

### Filosofi Pembagian Kerja

> **"Saya sebagai arsitek & QA, AI sebagai developer."**

### Bagian yang 100% Generated by AI

- Seluruh struktur HTML (`index.html`, `detail-product.html`, `detail-invoice.html`, `detail-order.html`, `admin.html`, `about-us.html`, `contact.html`, `help-center.html`, `order-guide.html`, `feedback.html`, `privacy-policy.html`, `terms-of-service.html`, `disclaimer.html`, `404.html`)
- Logic JavaScript kompleks: sistem stok dengan bracket syntax `[...]` dan pipe syntax `Label|Value`, kalkulasi Biaya Layanan & Diskon, auto-cancel pesanan, session management
- Modul JavaScript terpisah: `config.js`, `utils.js`, `frontend-features.js`, `admin-features.js`, `theme-engine.js`
- Halaman preview invoice (`detail-invoice.html`) — konfirmasi pesanan dengan ringkasan lengkap sebelum pembayaran
- Cloudflare Worker (`worker.js`) — sekitar **1.500+ baris** kode backend serverless
- Firebase Security Rules (`firebase.rule`)
- Sistem tema musiman dengan efek partikel (salju, kembang api, hati)
- Admin Panel dashboard lengkap dengan fitur CRUD, filter, dan live search
- Sistem Discord Webhook untuk hosting gambar produk gratis
- UI/UX polish: ikon SVG konsisten (tanpa karakter teks), desain responsif, animasi halus
- Semua file dokumentasi (README, architecture_and_security.md, documentation.md, MAINTENANCE_GUIDE.md)

### Bagian yang Membutuhkan Pendekatan Manual

- **Desain konsep & arsitektur awal** — saya yang menentukan stack (Firebase + Cloudflare + Mayar), saya jelaskan ke AI sebagai konteks
- **Testing & QA lapangan** — saya yang uji alur pembayaran nyata dengan Mayar sandbox, lalu laporkan bug ke AI untuk difix
- **Iterasi prompt yang tepat** — menemukan cara menjelaskan fitur kompleks (misalnya "stok bracket syntax") ke AI agar hasilnya sesuai visi
- **UI review & polish direction** — memastikan tombol, ikon, dan elemen UI terlihat profesional (misal: mengganti teks arrow `←` dengan SVG yang proper)
- **Konfigurasi environment** — setup API Key Mayar, Firebase project, Cloudflare environment variables
- **Review keamanan** — meminta AI audit keamanan, lalu saya validasi temuannya secara manual
- **Deployment & DNS** — proses deploy ke Cloudflare Pages dan konfigurasi custom domain tetap dilakukan manual

### Alur Vibecoding

```
Ide / Masalah
    ↓
Saya jelaskan ke AI dengan konteks yang detail
    ↓
AI generate kode (HTML/JS/CSS/Worker)
    ↓
Saya test di browser / Mayar sandbox
    ↓
Jika ada bug/mismatch → Saya jelaskan ke AI → AI fix
    ↓
Iterasi sampai fitur selesai & production-ready
    ↓
Deploy ke Cloudflare Pages
```

Proses ini berlangsung sangat cepat — fitur yang normalnya butuh **1-2 minggu** coding manual bisa selesai dalam **1-2 hari** dengan vibecoding.

ArufKuy Store bukan sekadar proof-of-concept — ini adalah **aplikasi production-ready** yang sudah digunakan untuk transaksi nyata. Seluruh alur dari:

1. Pelanggan memilih produk dari katalog
2. Melihat detail produk dan memilih varian
3. Mengisi data pembeli dan mengklaim kupon diskon
4. Mereview pesanan di halaman konfirmasi (preview invoice)
5. Membayar via Mayar (e-wallet / bank transfer)
6. Stok otomatis terkirim ke email pelanggan
7. Pelanggan tracking status pesanan via link aman
8. Admin memantau dari dashboard

...berjalan **end-to-end tanpa intervensi manual**, dibangun hampir seluruhnya dengan bantuan AI dalam proses vibecoding yang produktif.
